# 英雄批量购买器 功能升级计划 (v2.0)

本项目旨在为当前的批量购买器追加四大核心功能：多维度筛选、购买清单持久化与快捷购买、防爆破余额校验、以及现代化视觉交互升级。由于涉及大量的UI重制和核心逻辑重构，需要您的审阅以确定技术选型的细节设计。

## 🎯 Proposed Changes

---

### Phase 1: 引入轻量化现代化 UI 框架 (基础建设)
为彻底摒弃原生 `confirm()` / `alert()` 原生弹窗的丑陋外观，增强整体质感（结合“WOW”设计理念要求高级与动态），我们将接入 **Element Plus** 并在前端建立基于 Dark Mode 的主题结构。
- **依赖安装：**
  - 在前端 `packages/frontend` 安装 `element-plus` 以及相应的自动导入插件或直接全量配置。
- **UI 组件重构：**
  - 使用 `ElMessage` 替代现有的所有文字级别的错误或成功提示。
  - 使用 `ElMessageBox` 或自定义的 `ElDialog` 对话框重构“购买确认”、“列表保存询问”等重要交互，提供友好的“确认/取消”操作和漂亮的面包屑动画。

### Phase 2: 余额防爆破安全校验
- **后端拦截：** 无需，主要在前端完成感知。
- **前端拦截 (`stores/lcu.ts`)：**
  - 当触发购买时，立刻进行总计 `estimatedCost` 预估与 `wallet.ip` 及 `wallet.rp` 的对比。
  - 若余额不足，立刻抛出直观的 `ElMessage.error` 或弹窗阻止网络请求，避免向 LCU 发送无效请求导致的锁死和限流报错。

### Phase 3: 多维度高级筛选功能
在顶栏工具栏追加一组高级筛选控件，支持以下筛选：
1. **角色/职位筛选：** 
   基于接口原生的 `tags` 字段（法师 Mage，战士 Fighter，坦克 Tank，刺客 Assassin，射手 Marksman，辅助 Support）建立预设的快捷选择按钮，与国服商城保持一致。
2. **价格筛选：** 
   追加区间筛选或预设选项（如 `<3150精粹`, `4800精粹`, `6300精粹`等），让用户快速圈定可消费目标范围。
3. **状态结合：** 
   高级筛选与原有的 “搜索框” 和 “货币类型” 形成联合查询（Computed）。

### Phase 4: “购物车模板清单”跨账号应用 (基于反馈已修改)
实现“批量代买模板”的心智模型，适合跨账号使用。
- **本地持久化存储体系：** 引入 `localStorage` 持久化存储数据结构 `Record<string, { name: string, championIds: number[], date: number }>`。保存行为完全独立于购买行为之外。
- **保存模板入口改造：**
  - 在底部的结算浮签条中（除了包含已选择的数量和价钱外），**在“立即购买”按钮旁边紧挨着放置一个“保存清单”按钮**。
  - 用户勾选好一批英雄后，点击即可将其命名（如：“必买上分池”、“小号冲分合集”）并保存到本地。
- **清单管理与一键载入：**
  - 增加一个「我的清单/模板管理」入口（使用 Drawer 右侧滑出抽屉或 Dialog 对话框）。
  - 在管理界面中，用户可以看见过去保存的清单。
  - 点击“应用模板”，当前界面会自动将满足该清单的未拥有英雄勾选进入购物车，以便直接购买。此时如果遇到多选或者修改需求，亦可直接在此界面再修改选中状态，并在结算浮签条选“保存清单”（同名覆盖实现修改逻辑）。
